I'm a **software developer by passion**, currently an undergrad at **National Institute of Technology Durgapur**. Somewhere between solving complex logic puzzles and fixing merge conflicts, I found my absolute obsession.

## 🚀 Epic Builds

> *Four different problems, four different builds. None of these is "the main character," they're just things I wanted to exist.*

<table>
<tr>
<td width="50%">

### ⚙️ NodeBase — AI Workflow Engine
`React Flow` `Next.js` `tRPC` `Inngest` `Prisma`

A drag-and-drop, no-code builder for stitching together multi-step LLM pipelines. Event-driven execution with topological sorting handles complex DAGs without hitting serverless timeouts, wrapped in a multi-tenant setup with RBAC and encrypted API keys.

🔗 [Repo](https://github.com/Ankit-Gope007/NodeBase) · [Live](https://node-base-ten.vercel.app)

</td>
<td width="50%">

### 🧠 Campus Companion
`RAG` `Mistral-7B` `Embeddings` `FastAPI`

An AI campus assistant that actually knows whether to hit a database, search documents, or ask an LLM. RAG pipeline over embeddings + Mistral-7B for grounded answers instead of generic chatbot fluff. 

🔗 [Repo](https://github.com/nishgit17/CAMPUS_COMPANION)

</td>
</tr>
<tr>
<td width="50%">

### 🧾 Audition Portal
`Full-Stack` `Auth` `RBAC`

Built so a college audition season doesn't run on spreadsheets and panic. Role-based dashboards for coordinators, structured submissions for students — survives **400–500+ users annually** without breaking a sweat.

</td>
<td width="50%">

### 🎓 Academia Validator
`OCR` `Immutable Ledger` `RAG`

Turns a stack of academic documents into verifiable, tamper-proof records. OCR pulls the data, an immutable ledger keeps it honest, and a RAG layer lets you actually query and verify records instead of squinting at PDFs.

🔗 [Repo](https://github.com/Ankit-Gope007/ocr-working) · [Live](https://ocr-working.vercel.app)

</td>
</tr>
